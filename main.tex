\begin{abstract}
This paper reports a reproducible numerical-analysis benchmark for block-based image decryption using the 24-image Kodak lossless benchmark set. Each image is resized to $128 \times 128$, partitioned into $8 \times 8$ blocks, encrypted with dense and sparse key matrices, corrupted with additive Gaussian noise, and then decrypted by Gaussian Elimination with partial pivoting, LU Decomposition, Gauss-Seidel with relaxation, GMRES, and a NumPy linear-solve baseline. The study emphasizes numerical robustness, reconstruction quality, convergence behavior, and runtime under controlled perturbations. The generated artifact bundle includes Kodak manifests, wall-of-decryption figures, convergence traces, delta-versus-baseline summaries, and LaTeX-ready tables for direct inclusion in the manuscript.
\end{abstract}

\section{Introduction}
Matrix-based image encryption can be expressed as a repeated sequence of linear systems. For a vectorized plaintext block $x \in \mathbb{R}^{64}$ and key matrix $K \in \mathbb{R}^{64 \times 64}$,
\[
    c = Kx,
\]
and decryption under transmission noise is modeled as
\[
    c' = c + \epsilon, \qquad K\hat{x} = c'.
\]
The quality of the recovered image therefore depends on the numerical behavior of the solver used to invert the block system, especially when the key structure is sparse or when the encrypted blocks are perturbed by stochastic noise.

This work is a numerical analysis study of matrix-based decryption workflows, not a security proof of modern cryptographic primitives. The goal is to compare robustness, runtime, and reconstruction fidelity across direct, iterative, and hybrid solvers under the same Kodak-only benchmark protocol.

\subsection{Pipeline}
The experimental workflow follows the pseudocode-driven plan used for the notebook update.
\begin{enumerate}
\item Convert each benchmark image to grayscale and resize it to $128 \times 128$.
\item Partition the image into $8 \times 8$ blocks, yielding $64$-dimensional linear systems.
\item Generate a dense key matrix and a sparse key matrix.
\item Encrypt each block and inject additive Gaussian noise before decryption.
\item Recover each block with multiple solvers and compare numerical quality and cost.
\end{enumerate}

The sparse key is intentionally used alongside the dense key because solver behavior can change substantially when the coefficient matrix contains many zeros or near-decoupled interactions.

\section{Methods}
\subsection{Solvers}
The notebook implements the following methods:
\begin{itemize}
\item \textbf{Gaussian Elimination with partial pivoting}: direct method used as the classical baseline for row-wise elimination stability.
\item \textbf{LU Decomposition (Doolittle)}: factorization-based method that amortizes the matrix factorization across repeated block solves.
\item \textbf{Gauss-Seidel with relaxation}: iterative method used to study convergence and noise sensitivity in repeated fixed-point updates.
\item \textbf{GMRES}: Krylov-subspace hybrid method added to match the target plan and to provide a modern iterative reference.
\item \textbf{NumPy solve}: baseline reference built on dense linear algebra routines, used for delta-versus-baseline analysis.
\end{itemize}

\subsection{Independent Variables}
The notebook varies two main factors.
\begin{itemize}
\item \textbf{Matrix type}: dense and sparse key matrices.
\item \textbf{Noise level}: Gaussian noise with $\sigma \in \{0.00, 0.01, 0.03, 0.07, 0.12\}$.
\end{itemize}

Each condition is repeated three times across all 24 Kodak images. The run configuration and machine metadata are stored in \texttt{out/config.json} and \texttt{out/run\_info.json}.

\subsection{Evaluation Metrics}
The paper uses the artifacts generated by the notebook and post-processing scripts.
\begin{itemize}
\item PSNR and MSE for reconstruction quality.
\item Relative error and relative residual for numerical accuracy.
\item Median runtime and interquartile range for efficiency.
\item Success rate for robustness under each condition.
\item Gauss-Seidel convergence traces for iterative behavior.
\item Delta versus \texttt{np\_solve} for baseline-relative comparison.
\end{itemize}

\subsection{Interpretation}
The quantitative tables and figures are designed to answer four questions.
\begin{enumerate}
\item How much reconstruction quality is lost as Gaussian noise increases?
\item Does sparse structure help or hurt solver behavior under the chosen diagonal boosts?
\item How large is the runtime gap between direct, factorization-based, iterative, and hybrid methods?
\item How far do the custom solver implementations deviate from a dense linear-solve baseline?
\end{enumerate}

The artifact set generated by the notebook is intended to support those questions directly rather than through manually copied statistics.

\section{Discussion}
Three practical observations motivate the current design.

First, LU decomposition is expected to remain highly competitive whenever the same key matrix is reused across many blocks, because the factorization cost is paid once and the subsequent solves are comparatively cheap.

Second, Gauss-Seidel and GMRES provide complementary iterative viewpoints. Gauss-Seidel exposes convergence behavior directly through residual traces, while GMRES offers a hybrid Krylov-based reference for the same noisy block systems.

Third, the dense-versus-sparse comparison matters numerically even outside classical conditioning summaries. Two matrices can have similar condition numbers and still differ in residual propagation, convergence behavior, and runtime profile once the solver and noise process are fixed.
